# Service Monitoring Utility

## INTENT
Provide real-time visibility into the health, resource usage, and recent errors of the Upsilon stack services during development and testing.

## THE RULE / LOGIC
- **Execution**: The utility is a standalone Go script (`scripts/watch_services.go`).
- **Data Source**: It parses a structured `.services.pids` file generated by `scripts/start_services.sh`.
- **Resource Monitoring**:
  - CPU usage (%) per service PID.
  - Memory usage (%) per service PID.
- **Error Tracking**:
  - Scan the last 1 minute of log files assigned to each service.
  - Trigger alerts for keywords: `error`, `panic`, `fatal`, `exception`.
- **Refresh Rate**: 2 seconds (default).
- **Interface**: Full-screen terminal dashboard with ANSI color coding.

## TECHNICAL INTERFACE (The Bridge)
- **Code Tag**: `@spec-link [[watch_services]]`
- **Source File**: `scripts/watch_services.go`
- **Dependencies**: `scripts/start_services.sh`, `scripts/stop_services.sh`, `.services.pids`, `ps` (system tool).

## EXPECTATION (For Testing)
- The utility accurately reflects the PIDs stored in `.services.pids`.
- CPU/Mem metrics update every cycle.
- Recent errors discovered in logs are highlighted in the dashboard.
- The utility exits cleanly on `Ctrl+C`.
